Objective/Outcomes:

By completing this module, you will be able to:

  • Apply creative constraints to generate unique, feasible project ideas
  • Refine vague ideas into clearly scoped, deliverable concepts

  • Identify a realistic Minimum Viable Product (MVP) feature set for short timeframes

1.1 Get Started

This challenge is about building something web-based ‚Äî apps, games, dashboards, quizzes, or interactive experiences ‚Äî using Artificial Intelligence (AI) as your co-creator.

The challenge:

  • Your output should be web-based, however depending on tooling you can certainly explore other formats
  • It should have a small, achievable MVP you can build in < 1 hour
  • It should be unique ‚Äî ideally something that speaks to you

You can:

  1. Embrace randomization to get a unique combination of scenario + audience + vibe + format to inspire you or enable exploration
  2. Bring your own idea and refine it to work with this exercise

  2. Start a new chat
  3. Select Web in the top toggle

Option 1 - Random Combo Generator

Looking for inspiration? Let's work with Copilot to help us generate some interesting ideas.

Interest Intaker

First we will start by bringing your own interests to the table.

Steps:

  1. Copy the prompt
  2. Submit the prompt to your new conversation in Copilot
  3. Iterate as needed
Prompt 
You are my fast "Interest Intake" assistant. Ask me 3 quick questions, then output a compact JSON Seed Profile on one screen.

Ask:
1)Top 5 interest tags (topics, hobbies, niches)?
2) 3 things to AVOID (topics, audiences, styles)?
3) Any audiences I especially care about (roles, communities, industries)?

After I answer output only as JSON for the interests, avoid and audience

Generate Ideas

Now that we have captured some of your input in a structured way, we will use that to help generate ideas.

Steps:

  1. Copy the JSON output from the previous prompt
  2. Modify the prompt by clicking Edit. Replace [PASTE JSON FROM PROMPT] with your JSON output
  3. Copy the modified prompt
  4. Submit the prompt
Prompt 
You are my 'Diverse Concept Generator'. Use the Seed Profile to generate 15 FRESH, beginner-level web prototype ideas. Do NOT mention difficulty. Do NOT ask the user to choose formats.

SEED PROFILE:
[PASTE JSON FROM PROMPT A HERE]

STRUCTURE:
Return TWO tables in this order (no preamble text):
1) "Interest-Aligned (10)" ‚Äî 10 rows based on the user's interests/target audiences.
2) "Wildcards (5)" ‚Äî 5 rows explicitly OUTSIDE the interests (still simple & buildable).

COLUMNS (for both tables):
Title | Concept | Audience | Goal | Aesthetic | Format | MVP Features | Stretch Goals

FORMAT RULES:
- Format must be one of: game, quiz, application, dashboard, data visualization or a format that can be built within the hard guardrails
- Include **all five formats at least once** across the 15 rows.
- No single format appears more than 5 times total.

HARD SIMPLICITY GUARDRAILS (apply to every row):
- Single page; no auth, no routing.
- No external APIs; use tiny hardcoded arrays only.
- Interactions limited to buttons, inputs, selects, checkboxes, toggles, drag-drop.
- State in memory (optional: localStorage).
- Suitable for bare HTML/JS or a minimal React starter.

DIVERSITY RULES (across all 15 rows):
- No duplicate Scenario or Audience across ALL rows.
- At least 12 distinct aesthetic descriptors total.
- Prefer styles that can be supported by the guardrails and can be supported by CSS: sleek, modern, kawaii, monochromatic, gradient, etc.
- Respect the "avoid" list everywhere.
- Titles: 2‚Äì4 words; no "Pro/Hub/App" clich√©s, but give catchy branded names.
- MVP scope: 3‚Äì4 bullets, each doable in ‚â§90 minutes under the guardrails.

QUALITY CHECK (do this silently before finalizing):
1) If any Scenario or Audience repeats, reroll that row.
2) If vibe words are generic, replace with more specific style words.
3) If an MVP bullet violates guardrails (APIs, auth, complex routing), replace it.

OUTPUT:
Return ONLY the two tables in the specified order. No extra commentary.

Option 2 - Refine Your Own Idea

If you already have a concept in mind, let's ensure it's optimized for our 60-minute timeline.

Note:

While this challenge focuses on web-based projects, you can explore other formats (like mobile apps or desktop applications) by customizing the prompts and using GitHub Copilot for a code approach. If you choose a non-web format, ensure you have the necessary development tools and dependencies installed on your system. This goes beyond the scope of this exercise.

Steps:

  1. Modify the prompt to replace the bracketed item [Describe your idea in 2‚Äì3 sentences] with your idea.
  2. Copy the prompt
  3. Submit the prompt to your new conversation in Copilot
  4. Iterate as needed
Prompt 
I am participating in a creative Prompt to Prototype challenge where the end goal is to build a web-based prototype in 4 modules:

1. Research
2. Product Requirements Document (PRD)
3. Branding & Logo
4. Prototype

Here is my idea: [Describe your idea in 2‚Äì3 sentences].

- Scenario (theme or context)
- Audience (target users)
- Vibe Words (2‚Äì3 adjectives for style and tone)
- Format (type of web experience ‚Äî app, game, dashboard, interactive toy, quiz etc.)
- Concept (short description)
- Goal (what it aims to achieve)
- Title (placeholder name)
- MVP Features (small, completable in <2 hours)
- Stretch Goals (2‚Äì3 enhancements)

Output the result in a table with these columns: Scenario | Audience | Vibe Words | Format | Concept | Goal | Title | MVP Features | Stretch Goals.
