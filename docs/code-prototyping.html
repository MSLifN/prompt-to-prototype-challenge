6.1 GitHub Copilot Free Tier Features

Before we start building, let's understand the three core Copilot features available in the free tier:

üó®Ô∏è Copilot Chat (Primary Tool)

What it does: Your AI pair programmer. Ask questions, generate code, get explanations.

How to open: Click the chat icon in the left sidebar or press Ctrl+Alt+I

Best for: Generating components, asking "how do I..." questions, planning features, debugging errors

‚ö° Code Completions (Auto-suggestions)

What it does: Suggests code as you type (appears in gray text)

How to use: Just start typing, press Tab to accept, Esc to dismiss

Best for: Writing functions, filling in repetitive code, completing patterns

‚úèÔ∏è Inline Chat (Quick Edits)

What it does: Modifies specific code you select in context

How to open: Select code, press Ctrl+I (Windows) or Cmd+I (Mac)

Best for: "Make this responsive", "Add error handling", "Fix this bug"

üí° Free Tier Workflow

With free tier, you'll work one file at a time using Chat and Inline Chat. This is actually great for learning‚Äîyou'll understand what each piece of code does as you build it!

6.2 Get Started

Time to bring everything together. You have your concept, market validation, PRD, and brand identity. Now let's build a working prototype that demonstrates your value proposition using feature-by-feature development.

This exercise encourages exploration. While we provide guidance, you're welcome to iterate and expand based on your creativity and goals.

6.3 Prerequisites

What You Need:
  • VS Code installed
  • GitHub Copilot extension (Free tier works!)
  • A web browser (Chrome, Firefox, Edge, Safari)
  • Your PRD and branding assets from earlier steps
‚úÖ No Terminal Required!

Both Option A and Option B work without npm, Node.js, or any command-line tools. Just open VS Code and start creating.

Getting Started:
  1. Option A (Code a Web Page): Create a new file called index.html in VS Code
  2. Option B (Create a Document): Create a new file with .md extension (e.g., my-project.md)
  3. Open Copilot Chat (click the chat icon in the sidebar or press Ctrl+Alt+I)
  4. You're ready to start building with AI assistance!

6.4 Add Your PRD

In a previous module, you generated a Product Requirements Document. Let's work with Copilot to refine it for rapid prototyping.

Steps:
  1. Locate your saved PRD you created. If you need to create a markdown file, select File > New File and name it PRD.md and copy your Product Requirements content to there.
  2. In Copilot Chat, drag your PRD.md file to the prompt box
  3. Use this prompt:
Prompt 
Attached is my Product Requirements document named PRD.md. Please refine and update this document to be optimized for vibe coding a lightweight client-side web application.

Make the purpose clear, identify core features for a simple prototype, outline key user flows, and ensure it supports building with limited time.

Update the document directly with your improvements.

6.5 Use GitHub Copilot Chat

GitHub Copilot Chat is your AI pair programmer. Use it to:

  • Generate component boilerplate
  • Write functions based on natural language descriptions
  • Debug issues and explain code
  • Refactor for better structure
  • Add styling and interactivity
Tips for Effective Prompts:
  • Be specific: "Create an HTML section that displays a list of tasks with checkboxes"
  • Reference your context: "Using the PRD features, build a form for [feature name]"
  • Iterate: Start simple, then ask Copilot to enhance or modify
  • Use inline chat: Press Ctrl+I (Windows) or Cmd+I (Mac) to edit code in place
Example Prompt 
Create an HTML quiz interface in a single file with:
- A question display area
- 4 multiple choice options as buttons
- A "Next Question" button
- JavaScript to track score
- Show results at the end
- CSS styling included in a style tag

6.6 Build Your MVP Features

Follow this workflow to implement your MVP:

Development Workflow:
  1. Start with structure: Create main component files for your app's sections
  2. Build one feature at a time: Reference your PRD and tackle MVP features in order
  3. Use Copilot for boilerplate: Let AI generate initial component code
  4. Test as you go: Check each feature works before moving on
  5. Add interactivity: Wire up buttons, forms, and state management
  6. Apply branding: Add your colors, logo, and typography
  7. Polish the UX: Add loading states, error messages, transitions
üéØ Keep it Simple

For a 1-hour MVP, stick to client-side only: no backends, no databases, no authentication. Use local state, mock data, or browser storage.

6.7 Apply Your Branding

Integrate the brand assets you created earlier:

Branding Integration:
  1. Colors: Add your hex colors to CSS variables or Tailwind config
  2. Logo: Import your logo file and add it to a header component
  3. Typography: Import web fonts (Google Fonts) or use system font stack
  4. Tone: Use copy that matches your brand voice
Example Prompt for Styling 
Update the App.css to use this color palette:
- Primary: #3b82f6
- Secondary: #8b5cf6
- Accent: #f59e0b
- Background: #f9fafb

Apply these colors throughout the app with a modern, clean design.

6.10 Troubleshooting

Copilot not suggesting anything

Check that Copilot is enabled (icon in status bar). Try adding a comment describing what you want, then press Enter.

Suggestions are off-target

Be more specific in comments or Chat. Provide context about your PRD, tech stack, or desired outcome.

Code has errors

Ask Copilot Chat: "Fix this error" and paste the error message. Or select the problematic code and use Inline Chat to debug.

HTML not rendering correctly

Make sure you saved the file. Refresh your browser. Check for missing closing tags. Use browser DevTools to inspect the HTML structure.

Styles not applying

Check that CSS is inside a <style> tag or linked correctly. Verify class names match. Use browser DevTools to inspect elements.

6.11 Test and Refine

Before finalizing your prototype:

  1. Test all features: Click through every button, form, and interaction
  2. Check responsiveness: Resize browser window to test mobile view
  3. Review console: Fix any errors or warnings in browser console
  4. Validate against PRD: Ensure all MVP features are implemented
  5. Polish UI: Fix alignment, spacing, and visual consistency
  6. Add loading states: Show feedback for any async actions
üêõ Quick Fixes

Use Copilot Chat to quickly address issues: "This button doesn't work, fix the onClick handler" or "Add error handling to this form submission."

6.12 Deploy Your Prototype

Make your prototype accessible online:

Quick Deployment Options:
GitHub Pages (Recommended for Single-File HTML):
  1. Push your index.html file to a GitHub repository
  2. Go to repository Settings ‚Üí Pages
  3. Select "Deploy from a branch" and choose main
  4. Your site is live at https://username.github.io/repo-name
Netlify (Drag and Drop):
  1. Go to netlify.com and sign up
  2. Drag your project folder onto the deploy area
  3. Get a live URL in seconds
  4. No build step required for static HTML!
Vercel:
  1. Sign up at vercel.com
  2. Import your GitHub repository
  3. Vercel serves static HTML automatically
  4. Get a live URL in minutes
üí° Pro Tip

All three platforms offer free tiers perfect for MVPs. Vercel and Netlify provide instant SSL and custom domains.

6.14 Success Criteria

How do you know when your prototype is complete? Use this checklist:

‚úÖ Minimum Success (MVP Complete)
  • Page opens in browser without errors
  • Core user action is functional (the main thing your app does)
  • UI is usable (user can complete the primary flow)
  • No console errors in browser DevTools
üéØ Good Success (Polished MVP)
  • All items from Minimum Success, plus:
  • Brand colors and styling applied
  • Responsive design (works on mobile)
  • Clear feedback for user actions (loading states, success messages)
  • README documentation created
üöÄ Great Success (Deployed & Shareable)
  • All items from Good Success, plus:
  • Deployed to a live URL (Vercel, Netlify, or GitHub Pages)
  • Multiple features from PRD implemented
  • Professional polish (animations, hover states, thoughtful UX)
  • Ready to share with stakeholders
‚è±Ô∏è Time Box Reminder

Remember: This is a 30-60 minute exercise. Aim for "Minimum Success" first, then iterate towards "Good" or "Great" if time allows. A working prototype is better than a perfect prototype that's unfinished!
