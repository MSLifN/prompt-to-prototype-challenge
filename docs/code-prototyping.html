6.1 Get Started

Time to bring everything together. You have your concept, market validation, PRD, and brand identity. Now let's build a working prototype that demonstrates your value proposition using feature-by-feature development.

This exercise encourages exploration. While we provide guidance, you're welcome to iterate and expand based on your creativity and goals.

6.2 Prerequisites

Requirements:
  • Node.js from nodejs.org
  • VS Code from code.visualstudio.com
  • GitHub Copilot account
Steps:
  1. Navigate to the starter repo
  2. Clone or download the repository:
    • Clone: Click Code, copy URL, run git clone [URL]
    • Download: Click Code > Download ZIP, extract files
  3. Open in VS Code: File > Open Folder
  4. Install dependencies: 
    npm install
  5. Start development: 
    npm run dev

6.3 Add Your PRD

In a previous module, you generated a Product Requirements Document. Let's work with Copilot to refine it for rapid prototyping.

Steps:
  1. Locate your saved PRD you created. If you need to create a markdown file, select File > New File and name it PRD.md and copy your Product Requirements content to there.
  2. In Copilot Chat, drag your PRD.md file to the prompt box
  3. Use this prompt:
Prompt 
Attached is my Product Requirements document named PRD.md. Please refine and update this document to be optimized for vibe coding a lightweight client-side web application.

Make the purpose clear, identify core features for a simple prototype, outline key user flows, and ensure it supports building with limited time.

Update the document directly with your improvements.

6.4 Use GitHub Copilot Chat

GitHub Copilot Chat is your AI pair programmer. Use it to:

  • Generate component boilerplate
  • Write functions based on natural language descriptions
  • Debug issues and explain code
  • Refactor for better structure
  • Add styling and interactivity
Tips for Effective Prompts:
  • Be specific: "Create a React component that displays a list of tasks with checkboxes"
  • Reference your context: "Using the PRD features, build a form for [feature name]"
  • Iterate: Start simple, then ask Copilot to enhance or modify
  • Use inline chat: Press Ctrl+I (Windows) or Cmd+I (Mac) to edit code in place
Example Prompt 
Create a React component for a quiz interface with:
- A question display
- 4 multiple choice options as buttons
- A "Next Question" button
- Track score in state
- Show results at the end

6.5 Build Your MVP Features

Follow this workflow to implement your MVP:

Development Workflow:
  1. Start with structure: Create main component files for your app's sections
  2. Build one feature at a time: Reference your PRD and tackle MVP features in order
  3. Use Copilot for boilerplate: Let AI generate initial component code
  4. Test as you go: Check each feature works before moving on
  5. Add interactivity: Wire up buttons, forms, and state management
  6. Apply branding: Add your colors, logo, and typography
  7. Polish the UX: Add loading states, error messages, transitions
üéØ Keep it Simple

For a 1-hour MVP, stick to client-side only: no backends, no databases, no authentication. Use local state, mock data, or browser storage.

6.6 Apply Your Branding

Integrate the brand assets you created earlier:

Branding Integration:
  1. Colors: Add your hex colors to CSS variables or Tailwind config
  2. Logo: Import your logo file and add it to a header component
  3. Typography: Import web fonts (Google Fonts) or use system font stack
  4. Tone: Use copy that matches your brand voice
Example Prompt for Styling 
Update the App.css to use this color palette:
- Primary: #3b82f6
- Secondary: #8b5cf6
- Accent: #f59e0b
- Background: #f9fafb
- Text: #1f2937

Apply these colors throughout the app with a modern, clean design.

6.7 Leverage Key Copilot Features

Inline Suggestions

As you type, Copilot suggests completions. Press Tab to accept, Esc to dismiss.

Chat Panel

Open with Ctrl+Shift+I to ask questions and get code snippets. Great for "How do I..." questions.

Inline Chat

Press Ctrl+I in a file to edit code in context. Ask Copilot to "refactor this," "add error handling," etc.

Generate Tests

Ask Copilot to generate tests for your functions to ensure reliability.

Explain Code

Select code and ask "Explain this" to understand what it does‚Äîhelpful for learning.

6.8 Common MVP Patterns

Here are some typical features you might build with example prompts:

Form with Validation

Prompt: "Create a contact form with name, email, and message fields. Add validation and show success message on submit."

Data List with Filtering

Prompt: "Create a component that displays a list of products with search/filter functionality using mock data."

Multi-Step Flow

Prompt: "Build a 3-step wizard with Next/Previous buttons and progress indicator. Store form data across steps."

Interactive Calculator/Tool

Prompt: "Create a budget calculator that takes inputs and displays real-time results with charts."

Results/Summary Screen

Prompt: "Design a results page that shows a summary card with key metrics and a call-to-action button."

6.9 Troubleshooting

Copilot not suggesting anything

Check that Copilot is enabled (icon in status bar). Try adding a comment describing what you want, then press Enter.

Suggestions are off-target

Be more specific in comments or Chat. Provide context about your PRD, tech stack, or desired outcome.

Code has errors

Ask Copilot Chat: "Fix this error" and paste the error message. Or select the problematic code and use Inline Chat to debug.

npm install fails

Ensure Node.js is updated. Try npm cache clean --force and reinstall. Check for version conflicts.

Styles not applying

Check CSS import statements. Verify class names match. Use browser DevTools to inspect elements.

6.10 Test and Refine

Before finalizing your prototype:

  1. Test all features: Click through every button, form, and interaction
  2. Check responsiveness: Resize browser window to test mobile view
  3. Review console: Fix any errors or warnings in browser console
  4. Validate against PRD: Ensure all MVP features are implemented
  5. Polish UI: Fix alignment, spacing, and visual consistency
  6. Add loading states: Show feedback for any async actions
üêõ Quick Fixes

Use Copilot Chat to quickly address issues: "This button doesn't work, fix the onClick handler" or "Add error handling to this form submission."

6.11 Deploy Your Prototype

Make your prototype accessible online:

Quick Deployment Options:
GitHub Pages (Recommended for Simple SPAs):
  1. Run npm run build to create production files
  2. Push your code to a GitHub repository
  3. Enable GitHub Pages in repository settings
  4. Deploy from the dist folder
Vercel (Recommended for React/Vite):
  1. Sign up at vercel.com
  2. Import your GitHub repository
  3. Vercel auto-detects Vite and deploys
  4. Get a live URL in minutes
Netlify:
  1. Sign up at netlify.com
  2. Drag-and-drop your dist folder
  3. Or connect GitHub for continuous deployment
üí° Pro Tip

All three platforms offer free tiers perfect for MVPs. Vercel and Netlify provide instant SSL and custom domains.
