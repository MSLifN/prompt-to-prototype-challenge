5.1 Get Started

Your MVP is the minimum version of your product that still delivers value to your audience.

In this exercise, you'll:

  • Focus only on your MVP features
  • Integrate your branding assets if possible

Use the fastest available tools to get it working in ~1 hour.

5.2 Choose a Build Approach

It's time to build. At this point, make a decision on how you will prototype.

  • No-Code Path: Ideal if you're not a developer or want to focus on design/interaction flow, or have limited time.
    Example: GitHub Spark or another no-code platform.
  • GitHub Copilot Path: Ideal if you're comfortable with code and want to customize deeply.

To build your prototype, you have two different routes you can explore: No-Code Tooling or AI-Assisted Tooling. You will need to select how you want to build. The below table outlines some of the common differences between both approaches. We highlight common tools and platforms here, but note 1P tools and platforms should be used.
